## 2) Core visual pattern seen in your references

Across the examples, high-performing designs repeat the same structure:

- **1 dominant headline block** (2–5 words, very bold, high contrast).
- **1 supporting line** (smaller but still readable on mobile).
- **Strong subject cutout** in center or lower-center.
- **Accent color system** (usually 1 accent + neutral white/black).
- **Directional stickers/shapes** (arrows, circles, labels) to guide attention.

## 3) Recommended font stack (modern + thumbnail-safe)

Use a controlled font set by role (instead of random font choice):

### A. Headline fonts (Primary)
- **Anton** — best for high-impact all-caps headlines.
- **Bebas Neue** — tall cinematic look, great for vertical compositions.
- **Archivo Black** — strong and clean, less extreme than Anton.
- **League Spartan (800/900)** — modern heavy sans with cleaner edges.

### B. Secondary fonts (Support lines / labels)
- **Montserrat (700/800)**
- **Inter (700/800)**
- **Manrope (700/800)**

### C. Accent / expressive font (use sparingly)
- **Permanent Marker** or **Caveat Brush** for words like “WRONG”, “NEW”, “FREE”.

### Usage rule
- Maximum **2 font families per thumbnail** (3 only for special campaign styles).
- Keep **headline in ALL CAPS** for mobile readability.

## 4) Text placement system (safe zones)

For 9:16 thumbnails (shorts/reels), define fixed zones:

- **Top zone (0–22%)**: main headline.
- **Mid zone (32–68%)**: subject face/body + stickers; avoid placing long text over eyes.
- **Bottom zone (76–96%)**: CTA/support statement.

For 16:9 thumbnails:

- **Top-left or top-center**: headline.
- **Right or left third**: subject.
- **Bottom strip**: supporting text or badge.

### Placement templates to standardize
1. **Hero Top + Bottom CTA**
   - Headline top, support line bottom.
2. **Stacked Center**
   - 2-line headline center-left + subject on right.
3. **Sticker Label**
   - Main headline + one angled yellow label for urgency words.

## 5) Typography treatment recipe (what makes references work)

### Headline treatment
- Weight: **800–900**
- Tracking: **-1% to -2%** for bold fonts
- Shadow: `rgba(0,0,0,0.55)` with medium blur
- Stroke (optional): subtle dark stroke for bright backgrounds

### Supporting text
- Weight: **600–800**
- Simpler shadow, less blur
- Keep line length short (3–6 words)

### Contrast rules
- Background dark → white headline + accent yellow/green/red.
- Background bright → dark headline + light glow/stroke.

## 6) Color systems from your references

Pick one palette per thumbnail:

- **Warning / Design critique**: `#FFD12E` + white + near-black.
- **Tech / Neon**: `#5CFF7A` + white + charcoal.
- **Sports / High energy**: red + white + black.
- **Creator Clean**: cyan/teal + white + skin-tone warm highlights.

Avoid using more than one bright accent at full intensity unless category demands chaos.

## 7) Elements library (the “extra things” that improve CTR)

Use 2–5 of these per thumbnail, not all at once:

- Arrow scribbles / curved arrows
- Sticker labels (angled rectangle tags)
- Circle/cross badges
- Icon tiles (app logos, UI chips)
- Glow rings / light flares
- Grain + subtle dust texture
- Tape/paper card effect

## 9) Implementation recommendations for this project

1. Add a **Font Style selector** in UI:
   - `Impact`, `Cinematic`, `Modern Clean`, `Marker Accent`.
2. Add a **Text Layout selector**:
   - `Top + Bottom`, `Stacked Center`, `Top + Sticker`.
3. Add an **Elements density selector**:
   - `Minimal`, `Balanced`, `Busy`.
4. Save selected style/layout in state and include in generation request.
5. Keep an auto-fit engine for each zone so text never clips.
6. Add per-layout safe margins (especially for 9:16).
